## GitHub Integration

Set up GitHub to automatically deploy documentation updates from your repository.

<Steps>
  <Step title="Create a Webhook" icon="hook">
    In your GitHub repo settings, navigate to Webhooks > Add webhook.
    
    Set the Payload URL to `https://api.example.com/webhooks/github`.
    
    Choose Content type: `application/json`.
  </Step>
  <Step title="Configure Events" icon="settings">
    Select events like "Pushes" and "Pull requests".
    
    Add a secret: `YOUR_WEBHOOK_SECRET`.
  </Step>
  <Step title="Test the Webhook" icon="play">
    Make a commit to your repo and verify the delivery in GitHub.
  </Step>
</Steps>

## Using Webhooks for Updates

Webhooks notify your systems of events like page updates or new versions. Configure them via the dashboard.

<Tabs>
  <Tab title="Payload Example" icon="code">
    ```json
    {
      "event": "page.updated",
      "data": {
        "path": "/introduction",
        "timestamp": "2024-10-15T10:30:00Z"
      }
    }
    ```
  </Tab>
  <Tab title="cURL Test" icon="terminal">
````bash
curl -X POST https://api.example.com/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "event": "test",
    "payload": {}
  }'
````
  </Tab>
</Tabs>

<CodeGroup tabs="JavaScript,Python">
```javascript
const response = await fetch('https://api.example.com/webhooks', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    event: 'doc.updated',
    path: '/integrations'
  })
});
```
```python
import requests

response = requests.post(
  'https://api.example.com/webhooks',
  headers={
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  json={
    'event': 'doc.updated',
    'path': '/integrations'
  }
)
```
</CodeGroup>

<Callout kind="alert">
Always use HTTPS for webhook endpoints and validate signatures with `YOUR_WEBHOOK_SECRET` to prevent unauthorized access.
</Callout>

## Exporting Documentation

Export your docs to various formats for sharing or archiving.

| Format | Endpoint | Parameters |
|--------|----------|------------|
| PDF | `/export/pdf` | `path`, `theme` |
| HTML | `/export/html` | `path`, `includeNav` |
| Markdown | `/export/md` | `path`, `recursive` |

<ParamField path="path" param-type="string" required="true">
  The documentation path to export, e.g., `/introduction`.
</ParamField>

<ParamField query="theme" param-type="string" required="false">
  Optional theme: `light` or `dark`.
</ParamField>
