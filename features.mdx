## Document Structuring and Folders

Create a logical hierarchy for your projects using folders and pages.

### Organizing Content

You define folders to group related documents. For example, structure a book project as:

```
projects/
  my-novel/
    chapters/
      chapter-1.mdx
      chapter-2.mdx
    notes/
      research.mdx
```

This setup mirrors your mental model, making navigation straightforward.

<Expandable title="Advanced Folder Tips" default-open="false">

  - Use descriptive names without special characters.
  - Nest up to 5 levels deep for complex projects.
  - Frontmatter in pages controls visibility and order.

</Expandable>

## Collaboration and Sharing Permissions

Invite team members to contribute while controlling access.

<Tabs>
  <Tab title="Editor Permissions" icon="edit-3">
    Editors can modify content and add comments.
    
    ```javascript
    // Example sharing config
    const permissions = {
      role: "editor",
      canEdit: true,
      canShare: false
    };
    ```
  </Tab>
  <Tab title="Viewer Permissions" icon="eye">
    Viewers access read-only versions.
    
    ```javascript
    // Viewer setup
    const permissions = {
      role: "viewer",
      canEdit: false,
      canDownload: true
    };
    ```
  </Tab>
</Tabs>

| Permission Level | Edit Content | Share Links | Download |
|------------------|--------------|-------------|----------|
| Owner            | Yes         | Yes        | Yes     |
| Editor           | Yes         | No         | Yes     |
| Viewer           | No          | No         | Yes     |

## Search and Version Control

Locate documents quickly and revert changes as needed.

You perform full-text searches across all content. Version control tracks edits automatically, allowing rollbacks.

<Steps>
  <Step title="Search Documents" icon="search">
    Enter keywords in the global search bar to find matches instantly.
  </Step>
  <Step title="View History" icon="clock">
    Click the history icon on any page to see change logs.
  </Step>
  <Step title="Revert Changes" icon="undo">
    Select a previous version and restore it with one click.
  </Step>
</Steps>

## Customization with Brand Colors

Tailor the interface to your style using simple configuration.

<CodeGroup tabs="JSON,TypeScript">
  ```json
  {
    "theme": {
      "primaryColor": "#0969DA",
      "secondaryColor": "#ffffff"
    }
  }
  ```
  ```typescript
  const config = {
    theme: {
      primaryColor: '#0969DA',
      secondaryColor: '#ffffff'
    }
  };
  ```
</CodeGroup>

Apply changes via the settings panel. Your brand color `{#0969DA}` updates buttons, headers, and accents site-wide.

<Callout kind="tip">
Test customizations in preview mode before publishing to ensure consistency across devices.
</Callout>
